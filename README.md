# Hybrid TSP Solver (Genetic Algorithm + Simulated Annealing)

This project implements a hybrid approach to solving the **Traveling Salesman Problem (TSP)** using Python. It combines the global search capabilities of a **Genetic Algorithm (GA)** with the local optimization power of **Simulated Annealing (SA)**.

## 🚀 Features

- **Genetic Algorithm Framework**:
  - **Tournament Selection**: Selects parents by running small "tournaments" within the population.
  - **Ordered Crossover (OX1)**: Ensures valid routes where every city is visited exactly once.
  - **Elitism**: The best route from the previous generation is preserved automatically.
- **Simulated Annealing Refinement**:
  - Every child generated by the GA undergoes a short Simulated Annealing process to improve it locally before joining the new population.
- **Configurable Maps**:
  - Load city coordinates from a JSON file.
  - Comes with 3 preset maps (`random_10`, `circle_12`, `sparse_15`).

## 📋 Prerequisites

- Python 3.6 or higher.
- Standard libraries for solver execution.
- Optional (for animation): [Manim Community Edition](https://docs.manim.community/).

## 🧱 Project Structure

- `tsp_config.py`: Centralized solver parameters (`GAConfig`).
- `tsp_core.py`: Core domain + hybrid GA-SA algorithm.
- `tsp_solver.py`: CLI entry point to run the solver in terminal.
- `tsp_visualization.py`: Reusable Manim helpers (cities and route drawing).
- `tsp_manim_scene.py`: Manim scene that animates route evolution.
- `maps.json`: City datasets.

## 🛠️ Installation & Usage

1.  **Clone the repository** (or download the files):
    - `tsp_solver.py`
    - `maps.json`

2.  **Run the solver**:
    ```bash
    python3 tsp_solver.py
    ```

### Visualize with Manim

1.  Install Manim (if needed):

    ```bash
    pip install manim
    ```

2.  Render the animation scene:

    ```bash
    manim -pql tsp_manim_scene.py TSPRouteEvolution
    ```

3.  Choose map and number of sampled routes (optional):

    ```bash
    TSP_MAP=mega_100 TSP_SAMPLES=6 manim -pql tsp_manim_scene.py TSPRouteEvolution
    ```

4.  **Select a Map**:
    The program will list available maps from `maps.json`. Enter the number corresponding to your choice.

    ```text
    Available Maps:
    1. random_10 (10 cities)
    2. circle_12 (12 cities)
    3. sparse_15 (15 cities)

    Select a map number: 2
    ```

5.  **View Results**:
    The program outputs the best distance found every 10 generations and the final best route at the end.

## ⚙️ Configuration

### Adding New Maps

You can add your own sets of cities by editing `maps.json`. Use the following format:

```json
{
  "maps": {
    "my_custom_map": [
      {"x": 10, "y": 20},
      {"x": 50, "y": 60},
      ...
    ]
  }
}
```

### Tuning Parameters

You can adjust the algorithm's behavior by modifying the constants at the top of `tsp_solver.py`:

- `POPULATION_SIZE`: Number of routes in each generation.
- `GENERATIONS`: Total number of evolution cycles.
- `MUTATION_RATE`: Probability of swapping two cities.
- `SA_INITIAL_TEMP` & `SA_COOLING_RATE`: Control the intensity of the local search.

## 🧠 Algorithm Details

1.  **Initialization**: A population of random routes is created.
2.  **Elitism**: The single best individual from the current generation is copied to the next generation unchanged.
3.  **Selection**: `k` individuals are randomly picked, and the one with the shortest distance becomes a parent.
4.  **Crossover**: Two parents create a child using Ordered Crossover (OX1) to preserve relative orderings.
5.  **Mutation**: Random swaps occur with a low probability.
6.  **Hybrid Improvement (SA)**: The child is not added immediately. It runs through a "cooling" schedule where small changes are made. Improvements are kept; worsening moves are accepted probabilistically to escape local optima.
7.  **Repeat**: This continues until the new population is full, then repeats for `GENERATIONS`.
